# Privacy Paper Dashboard

面向数据安全与隐私保护方向的论文筛选工作台。

项目使用 `FastAPI + SQLite + React + Vite` 构建，支持抓取近三年的候选论文、按状态管理、导入外部论文链接、翻译标题与摘要、生成 AI 解读、导出入选结果，并统一记录运行日志。

## 当前能力

- 抓取近三年的候选论文，支持按来源范围配置抓取会刊
- 支持顶会、期刊、预印本三类抓取源，内置 `arXiv cs.CR`
- 候选管理分为：
  - `本次抓取`
  - `候选池`
  - `待读论文`
  - `已选文章`
  - `历史库`
- 支持把论文标记为候选、待读、已选、已分享、驳回
- 支持批量状态变更，尤其适合处理“本次抓取”结果
- 支持导入单条或多条论文链接，自动抓取元数据并去重
- 导入论文会尽量识别真实来源，而不是显示为 `imported`
- 自动按规范化 URL 去重，避免历史库、候选池、抓取结果重复
- 翻译与 AI 解读已解耦：
  - 可单独执行“翻译标题/摘要”
  - 可单独执行“AI 生成解读”
- AI 解读结果按结构输出：
  - 核心观点
  - 关键内容摘录
  - 价值与边界
- 运行日志统一记录抓取、翻译、AI 解读动作与失败原因
- 列表页支持搜索、筛选，并记住各页面各自的筛选条件

## 运行方式

### 一键启动

Windows 下直接运行：

```powershell
.\start.bat
```

脚本会完成依赖安装、前端构建，并启动后端服务。

默认访问地址：

- 前端工作台：`http://127.0.0.1:8000/`
- 本地开发前端：`http://127.0.0.1:5173/`

### 手动启动

后端：

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r backend\requirements.txt
cd backend
..\.venv\Scripts\python.exe -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

前端开发：

```powershell
cd frontend
npm install
npm run dev -- --host 127.0.0.1 --port 5173
```

前端构建：

```powershell
cd frontend
npm install
npm run build
```

## 使用流程

1. 打开工作台。
2. 在“来源/关键词配置”中选择抓取范围、抓取条数、回溯天数和分数阈值。
3. 点击“开始抓取”。
4. 先在“本次抓取”中处理新增候选，再在“候选池”中做长期筛选。
5. 需要时执行：
   - `翻译标题/摘要`
   - `AI生成解读`
6. 把需要持续跟踪的论文标记为“待读”，把最终保留的论文标记为“已选”。
7. 在“已选文章”中导出 Markdown，必要时同步标记为“已分享”。

## 导入论文链接

支持在以下页面导入链接：

- 候选池
- 待读论文
- 已选文章
- 历史库

支持一次粘贴多条链接，建议每行一条。系统会：

- 抓取论文标题、摘要、来源、时间等元数据
- 自动识别常见学术站点来源
- 与历史库和现有列表按 URL 去重
- 已存在时更新元数据，不重复生成新条目

## 抓取与 AI 处理

### 抓取

抓取配置只负责“发现论文”，不再自动耦合翻译或 AI 解读。

可配置项包括：

- 抓取来源范围
- 每个来源抓取条数
- 回溯天数
- 最低入库分数
- 最大论文年龄

### 翻译

“翻译标题/摘要”只负责：

- 英文标题转中文标题
- 英文摘要转中文摘要

### AI 解读

“AI生成解读”只负责生成结构化解读，不负责翻译。

当前提示词要求 AI 输出基于原文的专业化摘要，强调真实性基础和可阅读性。

## 去重策略

系统会综合以下信息去重：

- 规范化 URL
- 标题规范化
- 历史分享记录
- 已驳回记录

同一 URL 的历史遗留文章会保留，但不会重复写入新的候选记录。

## 运行日志

运行日志页记录以下动作：

- 抓取
- 翻译
- AI 解读
- 翻译 + AI 解读

日志支持：

- 按动作筛选
- 按状态筛选
- 按关键字搜索
- 展开失败详情

## 抓取来源配置

抓取来源配置在 `sources.json` 中维护，前端设置页可直接编辑和保存。

当前前端支持按类别快捷选择：

- 全选当前
- 全选会议
- 全选期刊
- 全选预印本

## 技术栈

- 后端：`FastAPI`
- 数据库：`SQLite`
- 前端：`React`
- 构建工具：`Vite`

## 注意事项

- 本项目默认面向 Windows 本地运行环境
- AI API Key 与翻译密钥仅保存在浏览器本地 `localStorage`，不写入后端数据库
- 某些学术站点会对页面直抓有限制，系统会优先尝试 DOI / Crossref / 元数据方案降低失败率
